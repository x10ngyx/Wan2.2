# Sea-Style 三缓存 Threshold 组合实验报告

## 1. 实验目的

本实验评估 SeaCache 风格的三类 cache 在 Ali Prompt 1 上组合使用时的速度和质量表现：

- timestep cache：`seacache`
- block cache：SeaCache 风格 block-group cache，即 `block-group + sea_full_rel_l1 + accumulated`
- CFG cache：SeaCache 风格 CFG cache，即 `sea-threshold`

实验目标是扫描三类 cache 的不同 threshold 组合，观察 `threshold 组合 -> speedup / PSNR / reuse 统计` 的关系，找到质量优先、中等质量和速度优先区间的代表点。

比较指标包括：

- 推理加速效果：使用 compute-only inference elapsed time 计算 speedup。
- 视频质量：使用同 prompt、同 seed、同 shape 的 no-cache baseline，通过 FFmpeg PSNR 计算平均 PSNR。
- 缓存行为：记录 timestep / block / CFG 三类 cache 的 reuse / recompute 次数。

### 2.2 Cache 组合顺序

本实验保持项目约定的三缓存组合顺序：

1. CFG cache 位于最外层，先决定是否复用 CFG delta，从而跳过 uncond 分支。
2. 对每个实际运行的 `cond` / `uncond` 分支，先判断 timestep cache。
3. 只有 timestep cache 未命中时，才进入 block cache。
4. 只有 timestep cache 和 block cache 都未命中时，才真正执行 transformer blocks。

## 4. 简单结论

1. Sea-style 三缓存组合可以覆盖从近 baseline 到高加速低质量的宽范围：最高质量点约 `37.5 dB`，最快点达到 `5.644x`，但最快点 PSNR 只有 `11.914 dB`，质量不可用。

2. 高质量区域的加速收益较小。PSNR `>=35 dB` 的最快组合是 `sea_ts_0p10__sea_bg_0p10__sea_cfg_0p10`，只有 `1.025x`，说明在高质量约束下可复用空间有限，而且 cache 判断本身有开销。

3. 中等质量较有价值的代表点是 `sea_ts_0p20__sea_bg_0p20__sea_cfg_0p20`，达到 `1.496x / 24.898 dB`。这个点三类 cache 都有复用，但 block cache 复用很少，主要加速来自 timestep 和 CFG。

4. 当目标质量降低到 `19 dB` 左右，速度可以提升到约 `2.845x`；进一步降低到 `18 dB` 左右，速度可到 `3.575x`。但从 `18-19 dB` 继续追求速度时，PSNR 会快速下滑。

5. 从单维趋势看，threshold 从 `0.10/0.20` 往 `0.40/1.00` 增大时，平均 speedup 明显上升，但平均 PSNR 快速下降。较稳的组合区域集中在 `0.10-0.20`；`0.40` 以后更适合低质量高速度实验，不适合作为质量优先推荐。

总体看，Sea-style 三缓存组合完成了从 threshold 到质量/速度的有效数据闭环。对于 Ali Prompt 1，推荐后续重点围绕 `0.10-0.20` 的局部区域继续细扫；若目标是高质量，组合加速空间有限；若目标允许 `24-25 dB` 左右，`0.20/0.20/0.20` 是当前较清晰的平衡点。
